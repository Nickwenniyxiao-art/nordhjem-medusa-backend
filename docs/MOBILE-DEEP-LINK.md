# 移动端深链规范

## 概述

本章节用于定义移动端深链接（Deep Link / Universal Link / App Link）统一标准，保障跨渠道跳转一致性。  
应明确支持协议、平台能力、兼容行为和安全边界。  
建议使用“协议层 + 路由层 + 参数层 + 安全层”分层描述。  
需说明与营销投放、消息推送、站外落地页的协作关系。  
最佳实践是建立深链注册中心，避免冲突与重复定义。  

| 字段 | 模板 |
|---|---|
| Link 版本 | [v1.0] |
| 支持平台 | [iOS/Android/H5] |
| 默认回退页 | [https://...] |

- 检查清单：是否定义协议；是否定义回退逻辑；是否定义安全校验。

## 链接结构

本章节应定义深链标准格式，包括 scheme、host、path、query 与可选 fragment。  
需给出命名规范与大小写规则，防止客户端解析不一致。  
建议定义 URI 模板和参数字典，标注必填/可选字段。  
应限制敏感数据直接出现在 URL，优先使用短 token。  
对营销渠道参数（utm_*）应统一传递与落库口径。  

| 组成部分 | 规则模板 | 示例 |
|---|---|---|
| scheme | [appname] | [nordhjem://] |
| host | [module] | [product] |
| path | [resource/action] | [/detail] |
| query | [k=v&k2=v2] | [?id=123] |

1. 定义 URI 模板。  
2. 发布参数字典。  
3. 客户端对齐解析逻辑。

## 路由映射

本章节应建立深链到页面路由的映射规则，包含页面名称、前置条件、失败回退。  
需定义登录态要求、权限校验和版本兼容处理。  
建议维护统一映射表并自动化校验“链接可达率”。  
应说明跨端一致性策略（App/H5 同一资源 ID 对应一致内容）。  
对于下线页面，需定义废弃策略与兜底落地页。  

| deep link | 页面路由 | 前置条件 | 回退策略 |
|---|---|---|---|
| [app://product/detail?id=] | [ProductDetail] | [资源存在] | [商品列表页] |
| [app://order/detail?id=] | [OrderDetail] | [登录且有权限] | [登录页] |
| [app://promo/campaign?id=] | [Campaign] | [活动有效] | [活动首页] |

- 检查清单：是否定义鉴权；是否定义回退页；是否定义下线路由处理。

## 参数校验

本章节应定义参数合法性校验，包括类型、长度、枚举、签名与过期时间。  
需规定必填参数缺失时的处理方式和用户提示。  
建议对高风险操作使用一次性 token 与服务端二次校验。  
应记录参数校验失败日志并上报监控。  
最佳实践是提供参数校验单元测试样例。  

| 参数 | 规则 | 示例 | 失败处理 |
|---|---|---|---|
| id | [数字/UUID] | [12345] | [跳转错误页] |
| scene | [枚举值] | [push/ad/share] | [使用默认值] |
| token | [签名+过期] | [jwt/opaque] | [重新鉴权] |

1. 客户端初校验。  
2. 服务端复校验。  
3. 记录失败并告警。

## 兼容策略

本章节应定义不同 App 版本、不同平台及未安装 App 场景下的兼容策略。  
需说明最低支持版本和升级引导机制。  
建议对不支持路由的旧版本跳转到 H5 兜底页，并保留关键上下文参数。  
应明确 Universal Link/App Link 配置变更发布流程。  
最佳实践是通过自动化回归覆盖高频深链。  

| 场景 | 兼容策略 | 用户体验 |
|---|---|---|
| 未安装 App | [跳转下载页/H5] | [展示引导文案] |
| 旧版本 App | [升级提示/兜底页] | [保留参数] |
| 配置失效 | [回退 web 路由] | [可继续操作] |

- 检查清单：是否定义最低版本；是否定义兜底页；是否定义回归用例。
