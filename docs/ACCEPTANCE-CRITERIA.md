# 验收标准

## 验收范围

本文件用于 **Phase 1 MVP** 功能验收，覆盖从商品浏览到支付确认的核心交易链路。

## 功能验收条目

| 功能 | 验收条件 | 通过/未通过 |
| --- | --- | --- |
| 首页加载 | 首屏正常展示、无关键报错、可进入商品页 | 待验收 |
| 商品浏览 | 列表/详情可访问，价格与库存展示正确 | 待验收 |
| 购物车 | 加购、修改数量、删除功能可用，金额计算正确 | 待验收 |
| 结算 | 地址、配送、总额确认流程完整 | 待验收 |
| 支付 | Stripe 测试支付成功并写回订单状态 | 待验收 |
| 订单确认 | 支付后展示订单确认页与订单号 | 待验收 |

## 非功能验收条目

- 性能：LCP < 2.5s，关键页面可交互时间符合目标。
- 安全：基础安全扫描通过，无明显 XSS/CSRF 风险。
- 可用性：Upptime > 99%。
- 合规：GDPR 基线（隐私说明、数据最小化、删除能力）满足。

## 验收方法

- 自动化验收：CI（lint/type-check/test）+ E2E 测试覆盖关键链路。
- 手动验收：Owner 执行一次完整购物流程（浏览→加购→支付→确认）。
- 结果记录：在 PR 与发布记录中透明披露通过项、阻塞项与后续计划。
