# NordHjem 模块注册表

> 最后更新：2026-03-15

## 概述

本文档记录 NordHjem 后端的模块结构，帮助新成员快速了解系统组成。

## 模块总览

| 编号 | 模块名称                 | 路径                               | 状态 | 说明                                                                   |
| ---- | ------------------------ | ---------------------------------- | ---- | ---------------------------------------------------------------------- |
| M01  | Store API（店铺端接口）  | `src/api/store/`                   | 活跃 | 面向前台用户的购物车、结账、订单、账户、支付方式与补货订阅相关接口。   |
| M02  | Admin API（管理端接口）  | `src/api/admin/`                   | 活跃 | 面向运营后台的订单、商品、库存、财务、售后、工单、审计与安全相关接口。 |
| M03  | Brand 模块               | `src/modules/brand/`               | 活跃 | 品牌领域模块，包含品牌模型、服务与模块入口定义。                       |
| M04  | Ticket 模块              | `src/modules/ticket/`              | 活跃 | 工单/售后领域模块，包含工单与消息模型、服务与迁移。                    |
| M05  | Restock 模块             | `src/modules/restock/`             | 活跃 | 到货通知（补货订阅）领域模块，包含订阅模型、服务与迁移。               |
| M06  | Resend Notification 模块 | `src/modules/resend-notification/` | 活跃 | 负责通知发送能力（Resend 集成）的模块化服务封装。                      |
| M07  | Workflow 编排            | `src/workflows/`                   | 活跃 | 业务流程编排（如创建补货订阅、发送补货通知、弃单流程）。               |
| M08  | Subscriber 事件处理      | `src/subscribers/`                 | 活跃 | 监听订单、支付、库存、售后、登录等事件并执行异步处理。                 |
| M09  | Module Links（模块关联） | `src/links/`                       | 活跃 | 维护跨模块实体关联（如品牌-销售渠道、补货订阅-商品变体）。             |

## 模块间关系

主要调用/协作关系如下：

- Store API → Restock 模块：`store/restock-subscriptions` 路由触发补货订阅创建流程。
- Workflow 编排 → Restock 模块 / Resend Notification 模块：补货通知流程读取订阅并发送通知，再清理已通知记录。
- Admin API → Ticket 模块：后台工单与售后管理相关接口依赖工单域能力。
- Admin API / Store API → Subscriber 事件处理：核心业务操作触发事件后由订阅器执行通知、审计、统计与联动任务。
- Brand 模块 / Restock 模块 → Module Links：通过 links 定义跨域实体关系用于查询与数据组织。

## 模块文档模板

每个模块的详细文档（后续逐步补充）应包含：

1. **概述** — 模块职责和边界
2. **关键实体** — 数据模型和字段
3. **API 端点** — 路由和方法
4. **配置项** — 环境变量
5. **依赖关系** — 与其他模块的交互
6. **变更历史** — 重要改动记录

2. 使用上述模板填写内容
3. 更新本 README.md 的总览表格
4. 提交 PR
