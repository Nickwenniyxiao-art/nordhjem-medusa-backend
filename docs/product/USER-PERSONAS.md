# NordHjem User Personas

> 本文档定义 NordHjem 电商平台的核心用户画像，用于指导功能优先级、UI/UX 设计和营销策略。
> 关联 Issue: #655 | ROADMAP Ref: INFRA

---

## Persona 1: Nordic Design Enthusiast (Anna)

| 属性 | 描述 |
|------|------|
| 姓名 | Anna K. |
| 年龄 | 28-38 |
| 职业 | 设计师 / 创意行业从业者 |
| 所在地 | 丹麦哥本哈根 / 北欧城市 |
| 收入水平 | 中高（年收入 40,000-65,000 EUR） |
| 技术熟练度 | 高，习惯线上购物 |

### 行为特征

- 关注设计感和材质，愿意为高品质北欧设计支付溢价
- 购买前会仔细查看产品图片、材质说明和尺寸参数
- 经常浏览 Instagram/Pinterest 获取家居灵感
- 偏好简洁、留白多的购物界面，讨厌弹窗和干扰
- 购物频率：每季度 1-2 次大件，偶尔购买小件装饰品

### 痛点

- 很多家具电商网站设计过于杂乱，不符合北欧美学
- 难以在线判断家具的真实质感和尺寸适配性
- 跨境购买时运费不透明，结算时才发现额外费用
- 退货流程复杂，尤其是大件家具

### 购买动机

- 追求"少即是多"的生活方式，选品精而不多
- 信任北欧品牌的设计语言和制造品质
- 希望一站式购买完整的北欧风格家居方案

### 对 NordHjem 的期望

- 高质量产品图片和详细的材质/尺寸信息
- 简洁的购物流程，不超过 3 步完成结算
- 丹麦克朗（DKK）本地化定价和配送
- 清晰的退货政策（14 天退货期）

## Persona 2: International Home Decorator (James)

| 属性 | 描述 |
|------|------|
| 姓名 | James M. |
| 年龄 | 32-45 |
| 职业 | 科技行业 / 金融从业者 |
| 所在地 | 美国纽约 / 伦敦 / 西欧城市 |
| 收入水平 | 高（年收入 80,000-150,000 USD） |
| 技术熟练度 | 高 |

### 行为特征

- 刚搬新家或翻新公寓，集中采购家具
- 偏好北欧/极简风格，但不一定了解具体品牌
- 通过 Google 搜索"Scandinavian furniture online"找到 NordHjem
- 价格敏感度中等，关注品质和设计，但会比价
- 购买决策周期较长（1-4 周），会多次浏览后下单

### 痛点

- 不确定从海外网站购买是否可靠（信任门槛）
- 国际运费和关税不透明
- 汇率波动影响实际支付金额
- 语言障碍（部分北欧网站无英文界面）

### 购买动机

- 追求与众不同的家居风格，区别于 IKEA 的"标准化"
- 愿意为"正宗北欧设计"支付国际运费
- 作为生活品质提升的投资

### 对 NordHjem 的期望

- 完整的英文界面和 USD/EUR/GBP 定价
- 国际配送信息透明（运费、关税、时效）
- Stripe 安全支付，支持主流信用卡
- 品牌故事和设计师背景增强信任感

## Persona 3: Small Business Buyer (Maria)

| 属性 | 描述 |
|------|------|
| 姓名 | Maria L. |
| 年龄 | 35-50 |
| 职业 | 室内设计师 / 小型酒店/咖啡店经营者 |
| 所在地 | 欧洲各地 |
| 收入水平 | 商业采购预算（5,000-30,000 EUR/年） |
| 技术熟练度 | 中等 |

### 行为特征

- 为客户项目或自有商业空间批量采购
- 需要查看产品的商业适用性（耐用度、可清洁性）
- 关注批量定价和 B2B 服务可能性
- 可能需要发票和税务文档

### 痛点

- 大多数 DTC 电商不提供批量询价或商业发票
- 需要确认产品在商业场景下的使用寿命
- 项目有时间线压力，需要确定交付时间

### 购买动机

- 为商业空间打造北欧风格氛围
- 客户指定要北欧设计风格的家具
- 建立长期供应商关系

### 对 NordHjem 的期望

- 产品信息包含商业适用指标
- 未来支持批量询价功能（当前为 Out of Scope，Phase 4+）
- 可靠的订单追踪和交付时间承诺

> **注意**：Maria 代表潜在的 B2B 需求。当前 NordHjem 定位为 DTC，B2B 批发分销明确在 PROJECT-SCOPE 的范围外。此画像用于记录未来扩展方向，不影响当前开发优先级。

## Persona 4: Platform Admin (Owner/运营)

| 属性 | 描述 |
|------|------|
| 姓名 | NordHjem Owner |
| 角色 | 平台所有者 + 日常运营 |
| 技术熟练度 | 中等（非专业开发者） |

### 行为特征

- 使用 Medusa Admin Panel 管理商品、订单和库存
- 需要快速响应新订单和客服工单
- 关注销售数据和库存状态
- 单人运营，需要高度自动化

### 痛点

- Admin 面板功能尚未 100% 完善（当前 60%，R-P3-02）
- 需要手动处理退款和异常订单
- 缺少直观的销售报表和库存预警

### 对 NordHjem 的期望

- 简洁高效的 Admin 操作界面
- 自动化的订单状态通知（Resend 邮件 + Telegram 告警）
- 库存低时自动提醒（restock module）
- Metabase BI 数据看板（R-P1-17）

## User Journey Map (核心购物流程)

```
发现 --> 浏览 --> 选择 --> 加购 --> 结算 --> 支付 --> 确认 --> 收货
  |       |       |       |       |       |       |       |
Google  分类/   详情页  购物车  地址+   Stripe  订单   物流
/社媒   搜索   图片+   管理   运费+   支付    确认   追踪
        筛选   规格          税费    回调    邮件
```

### 各阶段关键指标

| 阶段 | 关键指标 | 目标 |
|------|---------|------|
| 发现 | 自然搜索流量、社媒引流 | SEO 优化（R-P1-21, R-P3-05） |
| 浏览 | 页面加载时间 | LCP < 2.5s |
| 选择 | 产品详情页停留时间 | > 30s |
| 加购 | 加购率 | > 5% |
| 结算 | 购物车放弃率 | < 70% |
| 支付 | 支付成功率 | > 95% |
| 确认 | 订单确认邮件送达率 | > 99% |
| 收货 | 退货率 | < 10% |
