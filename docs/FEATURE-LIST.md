# 功能清单

## 功能地图

| 模块 | 子功能 | 用户对象 | 当前阶段 |
| --- | --- | --- | --- |
| 商品管理 | 商品 CRUD、分类、标签、图片 | Admin | P1 |
| 商品浏览 | 列表页、详情页、筛选、搜索 | Customer | P0 |
| 购物车 | 加购、删改数量、价格计算 | Customer | P0 |
| 结算 | 地址、配送、税费、订单确认 | Customer | P0 |
| 支付 | Stripe 支付、回调、失败重试 | Customer/System | P0 |
| 订单 | 下单确认、状态流转、查询 | Customer/Admin | P0 |
| 用户 | 游客下单、账号扩展（后续） | Customer | P1 |
| Admin | 商品/订单/库存管理 | Admin/Owner | P1 |
| SEO | 元信息、结构化数据、站点地图 | System | P2 |
| 监控 | 健康检查、日志、E2E 监控 | CTO/System | P2 |
| 增强能力 | 推荐、评论、收藏 | Customer | P3 |

## 功能优先级

- **P0**：核心购物流程（浏览、购物车、结算、支付、订单确认）。
- **P1**：Admin 管理能力（商品、库存、订单处理）。
- **P2**：SEO / 多语言 / 监控体系完善。
- **P3**：高级功能（推荐系统、评论体系、个性化体验）。

## 依赖关系

- 支付能力依赖 Stripe 集成与 Webhook 状态同步。
- 订单能力依赖购物车数据完整性与库存规则。
- Admin 操作依赖 Medusa 后端 API 与权限控制。
- 多币种与地区能力依赖 Region 配置及定价策略。

## 发布批次

- **Phase 1（MVP）**：商品浏览 + 购物车 + 下单 + Stripe 支付。
- **Phase 2（稳定化）**：运维基线、监控告警、发布流程固化。
- **Phase 3（扩展）**：SEO 增强、多语言完善、推荐/评论等增长功能。
