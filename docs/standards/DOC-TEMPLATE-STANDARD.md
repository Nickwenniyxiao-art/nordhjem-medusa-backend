# 文档格式模板标准 (DOC-TEMPLATE-STANDARD)

## 概述
本文档为项目文档库中的文档类型提供统一格式模板，目标是确保文档结构一致、门禁可校验、审计可追溯。模板内容基于 `docs/standards/DOC-LIBRARY.json` 自动对齐生成。

## 通用规则
- 所有文档必须包含 YAML 元数据头部（项目名称、创建日期、状态、负责人）。
- 所有文档必须使用 Markdown 格式。
- 标题层级统一为：`# 文档标题` → `## 章节` → `### 子章节`。
- 文件命名建议使用大写连字符（例如 `API-REFERENCE.md`）。
- 每类文档必须完整覆盖文档库定义的 `required_sections` 与 `required_metadata`。

## 通用元数据头部模板
```yaml
---
项目名称: <项目名>
创建日期: <YYYY-MM-DD>
状态: <草稿|审核中|已发布|已归档>
负责人: <角色/姓名>
---
```

## 模板使用指南
1. 从对应阶段选择目标文档类型模板。
2. 保留模板中的全部必填章节标题；仅可新增补充章节，不可删除必填章节。
3. 将 `<...>` 占位符替换为项目真实信息。
4. 如果文档适用范围不是 `all`，仅在匹配项目类型时启用该文档。
5. 若门禁（Gate）要求该文档为必须项，提交前需确认状态为“已发布”或按流程评审通过。
