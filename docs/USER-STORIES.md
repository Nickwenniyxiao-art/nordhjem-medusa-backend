# 用户故事

## 角色定义

- **顾客（Customer）**：浏览并购买家具产品的终端用户。
- **管理员（Admin/Owner）**：维护商品、库存、订单与运营策略。
- **系统（System/CTO）**：保障交易链路、稳定性与可观测性。

## 用户故事列表

1. 作为顾客，我想要按分类浏览商品，以便快速找到目标家具。  
2. 作为顾客，我想要在商品详情页查看尺寸和材质，以便确认是否适配家居空间。  
3. 作为顾客，我想要选择颜色/尺寸变体，以便购买符合需求的 SKU。  
4. 作为顾客，我想要把商品加入购物车，以便统一结算。  
5. 作为顾客，我想要在购物车修改数量，以便控制预算。  
6. 作为顾客，我想要在结算页填写配送地址，以便完成下单。  
7. 作为顾客，我想要使用 Stripe 支付，以便安全快捷地付款。  
8. 作为顾客，我想要支付后看到订单确认页，以便确认订单已创建。  
9. 作为顾客，我想要查看订单状态，以便了解配送进度。  
10. 作为管理员，我想要新增/编辑商品，以便保持目录更新。  
11. 作为管理员，我想要调整库存，以便避免超卖。  
12. 作为管理员，我想要更新订单状态，以便推进履约流程。  
13. 作为管理员，我想要处理退款申请，以便完成售后。  
14. 作为系统管理员（CTO），我想要监控关键交易链路，以便提前发现故障。  
15. 作为系统管理员（CTO），我想要在部署前自动执行 CI 检查，以便降低线上回归风险。  

## 验收条件

1. Given 顾客进入首页，When 选择分类，Then 展示对应商品列表。  
2. Given 顾客打开详情页，When 查看规格区域，Then 可见尺寸/材质/价格信息。  
3. Given 商品存在变体，When 顾客切换变体，Then 价格与库存显示同步更新。  
4. Given 顾客点击加购，When 操作成功，Then 购物车数量增加并提示成功。  
5. Given 购物车已有商品，When 修改数量，Then 小计与总价实时更新。  
6. Given 顾客进入结算，When 填写有效地址，Then 可进入支付步骤。  
7. Given 顾客完成支付信息，When Stripe 返回成功，Then 订单状态变为已支付/待履约。  
8. Given 支付成功，When 跳转完成，Then 显示订单号与摘要。  
9. Given 顾客访问订单页，When 输入有效凭据，Then 可查看当前状态。  
10. Given 管理员有权限，When 新增商品并保存，Then 店铺可见该商品。  
11. Given 管理员修改库存，When 保存后，Then 前台库存展示更新。  
12. Given 订单进入处理阶段，When 管理员更新状态，Then 状态变更写入系统并可追踪。  
13. Given 存在退款申请，When 管理员批准，Then 触发退款并记录流水。  
14. Given 系统运行中，When 关键链路失败，Then 监控触发告警。  
15. Given 有代码变更，When 提交 PR，Then lint/type-check/test 全部通过后才允许合并。
