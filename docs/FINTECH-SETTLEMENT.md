# 金融清结算规范

## 概述 — [详细说明该章节应包含的内容、格式要求和最佳实践示例]
本模板用于规范 [机构名称] 的交易清结算管理，覆盖资金清分、净额/全额结算、对账、差错处理与风险控制。  
文档应明确参与方（平台、通道、收单机构、合作银行、商户）及结算责任边界与时点定义。  
建议采用“时序图 + 账务分录 + 对账规则 + 异常剧本”组合呈现，确保技术与运营口径一致。  
应定义关键术语（清分、清算、结算、轧差、待结算、冻结款）并统一计时标准（时区/日切规则）。  
本节还应声明结算目标指标，例如准时结算率、差错率、未达账处置时效、资金占用天数。

## 结算流程 — [详细说明该章节应包含的内容、格式要求和最佳实践示例]
结算流程建议拆分为“交易入账 → 清分计算 → 费用计提 → 结算指令生成 → 出款执行 → 回执入账”六步。  
每个步骤应描述输入数据、校验规则、账务分录、失败重试策略以及人工介入条件。  
应区分全额结算与净额结算场景，并说明手续费、退款、拒付、保证金、税费等处理顺序。  
跨境或多币种场景需增加汇率来源、换汇时点、汇兑损益归属与对冲策略说明。  
建议定义结算截止时间 [Cutoff] 与补处理窗口 [补单窗口]，避免跨日差异扩大。  

**流程步骤（模板）**
1. [交易系统] 输出已清算交易明细并签名。
2. [账务系统] 执行清分引擎生成应结算明细。
3. [费用系统] 计算佣金、通道费、税费并回写。
4. [结算系统] 生成批次与付款指令。
5. [银行/通道] 返回执行结果与回执。
6. [总账系统] 完成最终入账与状态闭环。

## 对账机制 — [详细说明该章节应包含的内容、格式要求和最佳实践示例]
对账机制应覆盖内部账（交易、账务、总账）与外部账（银行、支付通道、商户）双向核对。  
建议采用“日清 + 周核 + 月结”多层对账节奏，并明确自动对账与人工复核分工。  
对账规则需定义主键（交易号/订单号/渠道流水号）与容差（金额、汇率、时差）标准。  
未达账、短款、重单、错单应分类管理，并记录初判原因、归属责任与预计解决时间。  
对账报告应包含差异规模、解决进度、逾期项、复发率等管理指标，用于持续改进。

## 异常处理 — [详细说明该章节应包含的内容、格式要求和最佳实践示例]
异常处理应建立标准剧本，覆盖结算失败、重复付款、金额不符、回执缺失、跨系统状态不一致等高频问题。  
每类异常需定义触发条件、影响评估、止损动作、责任归属、恢复步骤与对外沟通模板。  
建议按严重级别分级响应：P1（资金风险/监管影响）、P2（批量延迟）、P3（单笔差错），绑定响应 SLA。  
涉及客户资金风险时应优先执行“冻结相关批次 + 人工复核 + 管理层通知 + 事件记录”。  
异常关闭前必须完成根因分析（RCA）与预防措施（CAPA），并在 [复盘周期] 内验证改进效果。

## 风险控制 — [详细说明该章节应包含的内容、格式要求和最佳实践示例]
风险控制应围绕资金安全、操作风险、模型风险、合规风险建立“三道防线”控制体系。  
建议在系统层实施关键控制：双人复核、批次签名、限额阈值、异常熔断、敏感操作最小权限。  
资金层面应设置备付金监控、流动性预测、头寸预警与压力测试，确保极端情况下履约能力。  
运营层面应配置关键 KRI（如超时未结算金额、异常率、人工干预率）并建立日报/周报机制。  
同时应准备应急预案：通道故障切换、银行中断备援、手工应急结算流程和回滚策略。
