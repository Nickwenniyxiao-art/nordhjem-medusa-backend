# 用量计量规范

## 概述

本章节用于界定“什么是计量对象、为何计量、如何用于结算与风控”，建议先定义计量原则再列举应用场景。  
应明确计量事件与业务事件的关系，确保口径可审计、可复算、可回放。  
建议统一采用事件驱动模型：采集层、清洗层、聚合层、结算层分离。  
需定义时间标准（UTC）、幂等键策略和延迟容忍窗口，避免重复计费。  
最佳实践是将“计量变更”纳入发布流程，要求产品、财务、数据团队联合评审。

## 计量维度

本章节应列出需要计量的核心维度，如 API 调用次数、活跃用户数、存储占用、任务执行时长等。  
每个维度必须描述单位、采样频率、聚合周期、免费额度与超额规则。  
建议将维度划分为“结算维度”和“监控维度”，避免把监控指标直接用于收费。  
应定义维度权威来源（source of truth）和优先级，防止多系统冲突。  
建议提供维度字典表并标注数据拥有者，便于后续治理。

## 采集方法

本章节应说明日志埋点、SDK 回传、网关计数、数据库快照等采集手段及优缺点。  
需规定采集格式（JSON Schema）、必填字段、脱敏规则与签名校验机制。  
建议使用“实时 + 离线补偿”双路径：实时用于看板与限流，离线用于结算与对账。  
应设计去重策略（幂等键 + 时间窗口）和异常数据隔离队列。  
需明确数据延迟上限、丢失率阈值与补采流程，保证结算准确性。

## 结算口径

本章节应定义从原始事件到可计费账单的转换规则，包括聚合公式、舍入规则、免费额度扣减顺序。  
需明确对失败请求、内部调用、灰度租户、测试租户的处理口径。  
建议在口径文档中列出“示例账单推导”，支持财务和客户成功团队复核。  
应定义账期冻结时间（billing lock）与更正策略（冲正单/补差单）。  
最佳实践是维护口径版本化并提供变更对比，避免历史账单解释困难。

## 对账机制

本章节应规定内部对账（计量系统 vs 账单系统）与外部对账（客户账单 vs 使用明细）流程。  
建议建立 T+1 自动对账任务，输出差异清单并自动分级。  
需定义允许误差阈值、差异原因分类与处置 SLA。  
应提供对账证据链模板：原始事件、聚合结果、账单记录、调整单据。  
建议设置月度抽样审计和季度全量回归，保证长期一致性。
