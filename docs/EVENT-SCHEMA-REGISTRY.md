## 兼容性规则 — [详细说明该章节应包含的内容、格式要求和最佳实践示例]

本章节定义前向兼容、后向兼容与完全兼容策略，并映射到校验级别。
应明确允许变更（新增可选字段）与禁止变更（删除必填字段、改字段类型）。

需描述版本淘汰节奏与消费者迁移窗口。
最佳实践：每次 schema 变更自动触发兼容性测试与订阅方通知。
兼容性检查清单：

- [ ] 关键字段未破坏
- [ ] 默认值保持语义一致
- [ ] 消费者回归测试通过
- [ ] 迁移文档已发布
