# 可行性研究

## 研究背景

北欧家具市场持续线上化，用户对“风格化品牌 + 便捷下单 + 透明配送”接受度提升。DTC（直面消费者）模式可减少中间渠道成本、提高品牌控制力，并通过数据反馈快速迭代选品与营销。NordHjem 以北欧极简设计为差异化定位，具备通过内容化商品展示提升转化的潜力。

## 技术可行性

- **Medusa.js v2**：具备模块化电商能力（商品、订单、支付、区域、后台），适合中小团队快速交付。
- **Next.js（前端）**：SSR/混合渲染支持首屏性能与 SEO，适合电商目录场景。
- **PostgreSQL**：事务一致性与生态成熟，适配订单与支付状态管理。
- **Docker + Railway**：容器化部署简化环境一致性，利于低成本持续交付。
- **可维护性**：TypeScript 严格模式 + CI 基线（lint/type-check/test）可降低回归风险。

结论：技术栈成熟、生态可用，MVP 交付可行。

## 成本与资源评估

| 成本项 | 估算区间 | 说明 |
| --- | --- | --- |
| Railway 部署 | $5-20/月 | 按容器/数据库资源等级调整 |
| Stripe 手续费 | 2.9% + 30¢/笔 | 按交易量浮动，需计入毛利 |
| 域名与 SSL | $1-3/月（均摊） | 年付域名折算；SSL 通常平台托管 |
| AI 工具（Codex/GPT） | $10-30/月 | 用于开发、文档、测试自动化 |

资源评估：以 Owner + CTO(AI) 模式可支撑 MVP，需通过模板化流程减少手工运维。

## 风险与建议

### 主要风险

1. **单人团队瓶颈**：并行需求过多时容易交付延迟。
2. **Medusa v2 社区相对较新**：遇到边缘问题可能需要自主排障。
3. **支付与履约耦合**：异常流程（支付失败、超卖、退款）处理复杂。
4. **成本波动**：流量增长后托管与工具费用可能超预期。

### 建议

- 优先 MVP：先稳定“浏览-下单-支付-确认”闭环。
- 自动化一切：CI 校验、E2E 回归、文档与任务治理联动。
- AI 驱动开发：用 AI 加速文档沉淀、测试用例生成和问题定位。
- 分阶段扩展：先做可用，再做可扩展与可优化。
