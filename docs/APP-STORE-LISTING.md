# 应用商店上架清单

## 概述
本章节用于统一 iOS App Store 与 Google Play 上架流程，明确资料准备、审核沟通、发布节奏和责任分工。  
应列出版本范围（新上架/更新发布/紧急热修）和适用平台。  
建议采用“提交前清单 + 提交后跟踪 + 审核驳回处理”三段式流程。  
需说明与隐私合规、支付策略、账号体系的一致性要求。  
最佳实践是建立上架资产版本库，避免截图、文案和隐私条款不一致。

## 审核要求
本章节应列出平台审核常见要求：账号可用性、测试路径、支付合规、隐私权限说明、内容规范。  
需提供审核账号模板（用户名、密码、二步验证方式）并保证可登录。  
建议对高风险功能（登录、支付、用户生成内容）提供审核说明文档。  
应定义驳回场景处理流程与 SLA，减少反复拒审。  
最佳实践是维护“历史驳回库”，沉淀整改经验。

## 发布计划
本章节应定义发布时间、灰度策略、市场联动、客服值守和监控方案。  
需标注平台审核时长预估与缓冲时间，避免市场活动延期。  
建议按“候选版本冻结 → 提审 → 审核通过 → 分阶段发布”执行。  
应明确发布负责人、故障升级路径和回滚触发条件。  
对于重大版本，需设置 24-48 小时观察窗口。
