## 概述
本模板用于定义移动端无障碍（Accessibility）设计与验证标准，建议对齐 WCAG 2.2 与平台原生规范。
章节需覆盖视觉、听觉、操作、认知等多维需求，确保产品在不同能力用户群体中可用。
应明确适配范围：核心流程页面[列表]、组件库[名称]、动态内容与多媒体资源。
建议给出角色分工：设计、研发、QA、无障碍评审专家与业务 Owner。
流程步骤：需求评审 → 设计标注 → 开发实现 → 自动化检测 → 人工体验测试。
检查清单：是否有可达焦点、是否有语义标签、是否支持大字号和屏幕阅读器。
最佳实践：在需求阶段就引入无障碍验收标准，避免后置返工。

检查清单：是否支持深色模式、是否支持系统字体放大、是否避免仅颜色传达信息。
最佳实践：组件层封装默认可访问属性，减少业务页面重复实现成本。

## 交互要求
交互要求需涵盖焦点顺序、手势替代、语音播报文本、状态变化提示与错误恢复路径。
建议为每个交互组件定义“键盘/读屏/触控”三套行为，避免单一交互路径。
模板表：

| 场景 | 交互要求 | 无障碍属性 | 失败兜底 |
| --- | --- | --- | --- |
| [表单提交] | [错误项自动聚焦] | [accessibilityHint] | [语音提示+高亮] |
| [轮播图] | [可暂停/可跳过] | [aria-like label] | [静态列表替代] |

流程步骤：定义用户任务流 → 标注焦点路径 → 开发实现 → 读屏回归。
检查清单：是否避免焦点陷阱、是否保证动态更新可播报、是否支持横竖屏切换。

测试方法应结合自动化扫描与人工可用性测试，覆盖主流设备与系统版本。

4. 回归验证并记录缺陷闭环。
