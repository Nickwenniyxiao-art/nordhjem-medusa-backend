# NordHjem SLO/SLA 定义

## 服务等级目标 (SLO)

### 可用性
| 服务 | SLO | 允许停机时间/月 |
|------|-----|----------------|
| 后端 API | 99.9% | ~43 分钟 |
| 前端网站 | 99.9% | ~43 分钟 |
| 数据库 | 99.95% | ~22 分钟 |
| 支付服务 | 99.99% | ~4 分钟 |

### 性能
| 指标 | 目标 | 测量方式 |
|------|------|---------|
| API 响应时间 (P50) | < 200ms | Sentry/APM |
| API 响应时间 (P95) | < 500ms | Sentry/APM |
| API 响应时间 (P99) | < 1000ms | Sentry/APM |
| 前端 LCP | < 2.5s | Lighthouse/Web Vitals |
| 前端 FID | < 100ms | Web Vitals |
| 前端 CLS | < 0.1 | Web Vitals |
| 页面完整加载 | < 3s | Lighthouse |

### 可靠性
| 指标 | 目标 |
|------|------|
| 部署成功率 | > 95% |
| 回滚时间 | < 5 分钟 |
| MTTR (平均恢复时间) | < 30 分钟 |
| Error Rate | < 0.1% |

## 监控与告警

### 告警阈值
| 指标 | 警告 | 严重 |
|------|------|------|
| 可用性 | < 99.95% | < 99.9% |
| API P95 延迟 | > 500ms | > 1000ms |
| Error Rate | > 0.05% | > 0.1% |
| CPU 使用率 | > 70% | > 90% |
| 内存使用率 | > 80% | > 95% |

### 监控工具
- **可用性**: Uptime Kuma
- **性能**: Sentry Performance
- **前端**: Lighthouse CI + Web Vitals
- **告警**: Telegram Bot
