## 客户识别流程 — [详细说明该章节应包含的内容、格式要求和最佳实践示例]

建议将客户识别拆分为“准入校验 → 身份采集 → 实名核验 → 风险评级 → 审批开户”五阶段，并标注输入/输出。  
流程步骤示例：1) 收集基础信息；2) 采集身份证明与地址证明；3) 执行活体/人脸比对；4) 命中名单筛查；5) 生成尽调结论。  
每一步应定义责任角色 [角色名称]、操作系统 [系统名称]、超时阈值 [时限] 和升级路径 [升级规则]。  
需明确“拒绝开户”“要求补件”“人工复核”三类分支条件，确保前台、运营与合规口径一致。  
如涉及代理开户或远程开户，需增加授权文件真实性核验和双因子校验控制。

## 资料校验 — [详细说明该章节应包含的内容、格式要求和最佳实践示例]

资料校验应覆盖真实性、完整性、一致性、时效性四个维度，并按照客户类型配置差异化规则。  
个人客户最少应提供 [身份证件类型]、[地址证明类型]、[联系方式]；企业客户应补充 [营业执照]、[UBO 信息]、[授权证明]。  
建议引入多源校验：OCR 结构化提取、公安/工商接口核验、黑白名单比对、反欺诈设备指纹校验。  
对于高风险行业或跨境客户，要求增强尽调（EDD），包括资金来源说明、交易目的说明、受益所有人穿透材料。  
当系统校验与人工判断冲突时，应触发二线复核并记录冲突原因与最终裁决依据。

## 风险分层 — [详细说明该章节应包含的内容、格式要求和最佳实践示例]

风险分层应基于客户属性、地域、产品、渠道、交易行为建立评分卡模型，并定义分层阈值与复评频率。  
示例分层：低风险 [分值区间]、中风险 [分值区间]、高风险 [分值区间]，各层对应不同尽调深度与监控策略。  
评分因子建议包含：职业/行业敏感度、PEP/制裁关联、开户渠道可信度、设备风险、历史可疑行为。  
高风险客户必须执行增强尽调（EDD）并由 [审批角色] 二级审批后方可激活。  
应设置动态调级机制，当触发 [触发条件]（如短期大额异常）时自动提升风险等级并重新审核。  

**分层控制要点（模板）**

- [ ] 分层规则有版本号与生效时间。
- [ ] 分层阈值经合规与风控联合审批。
- [ ] 调级/降级均有审计轨迹与理由。
- [ ] 高风险客户复评周期不超过 [周期]。
- [ ] 分层结果已同步到交易监控系统。
