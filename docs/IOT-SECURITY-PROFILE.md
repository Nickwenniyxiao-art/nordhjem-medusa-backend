# IoT 安全配置基线

## 概述

本章节应定义 IoT 安全基线的目标、适用对象、威胁模型与合规框架。
建议覆盖设备、边缘、云、运维链路四个安全域。
应明确最低安全要求与例外审批流程，避免灰色配置。
最佳实践：安全基线版本化管理并与发布流程强绑定。

| 安全域 | 占位符 |
| --- | --- |
| 设备侧 | [Secure Boot/证书认证] |
| 通信侧 | [TLS1.3/mTLS] |
| 运维侧 | [最小权限+审计] |

## 身份认证

本章节需规定设备身份发放、注册认证、轮换吊销和失窃处置流程。
建议每台设备使用唯一证书与不可导出私钥。
应明确首次入网认证（bootstrap）和日常会话认证（mTLS/JWT）机制。
最佳实践：认证失败达到阈值后自动隔离设备。

| 场景 | 控制要求 | 失效策略 |
| --- | --- | --- |
| 首次注册 | [工厂证书+一次性令牌] | [超时失效] |
| 日常连接 | [mTLS 双向认证] | [失败5次锁定] |

## 密钥管理

本章节应定义密钥生命周期（生成、分发、存储、轮换、销毁）及职责分离。
建议使用 HSM/KMS 管理主密钥，设备侧仅保存会话或派生密钥。
应规定轮换周期、高危事件强制轮换与泄露应急预案。
最佳实践：密钥操作全部审计并最小化人工触达。

| 密钥类型 | 轮换周期 | 存储位置 | 审计要求 |
| --- | --- | --- | --- |
| 设备证书私钥 | [按证书有效期] | [安全元件] | [不可导出] |
| 平台 API Key | [90天] | [KMS] | [操作留痕] |

## 通信加密

本章节需规定传输层与应用层加密策略、算法白名单和证书校验规则。
建议默认 TLS1.3，禁用弱加密套件与过时协议版本。
应定义离线缓存数据加密和完整性校验，防止物理攻击泄露。
最佳实践：所有安全参数集中配置并支持远程下发。

| 控制项 | 基线要求 | 校验方式 |
| --- | --- | --- |
| 传输协议 | [TLS1.3+] | [握手探测] |
| 证书校验 | [OCSP/CRL + Pinning] | [连接前校验] |
| 本地存储加密 | [AES-256-GCM] | [启动自检] |

## 安全审计

本章节应定义审计日志范围、保存期限、告警规则与取证流程。
建议记录认证事件、密钥操作、配置变更、固件升级、安全策略命中。
应明确日志防篡改措施（签名链/远程备份）和最小留存周期。
最佳实践：安全审计结果纳入月度风险评审并闭环整改。

- 审计检查清单：
  - [ ] 关键安全事件已全量采集
  - [ ] 审计日志留存不少于 [180天]
  - [ ] 高危事件 [15min] 内触发告警

| 事件类型 | 告警级别 | 响应SLA |
| --- | --- | --- |
| [证书异常] | [P1] | [15min] |
