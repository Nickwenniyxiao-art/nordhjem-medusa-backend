# 产品需求文档（PRD）

## 产品目标

打造面向 DK 和 US 市场的北欧极简风格在线家具商店，提供从商品浏览到支付确认的完整购买体验，并建立可持续迭代的后台运营能力。

## 用户与场景

1. **场景 1：浏览商品目录**
   - 用户按分类、风格、价格区间浏览商品。
2. **场景 2：加入购物车并结算**
   - 用户选择规格后加购并完成 Stripe 支付。
3. **场景 3：管理订单（Admin）**
   - 管理员处理商品信息、库存、订单状态与售后。

## 功能需求

- 商品浏览与搜索：分类、关键词、过滤、详情页。
- 购物车管理：增删改查、价格明细与运费估算。
- Stripe 支付：支付意图创建、回调状态同步、失败重试提示。
- 订单追踪：下单成功页、订单状态查询、通知能力（邮件/站内后续扩展）。
- 多语言/多币种：至少支持英语界面与 EUR/USD 价格展示。
- Admin 管理后台：商品、库存、订单、退款基础操作。

## 非功能需求

- 性能：关键页面加载 < 3s，LCP < 2.5s。
- 可用性：月度可用性 >= 99.5%。
- 安全：支付链路 HTTPS，基础 XSS/CSRF 防护，最小权限访问。
- 合规：GDPR 合规（最小化采集、可追踪同意与删除流程）。

## 验收标准

- 完整下单流程可走通（浏览→加购→结算→支付→确认）。
- Stripe 测试支付成功并正确更新订单状态。
- Admin 可创建/编辑商品并管理订单状态。
- 在 DK/US 区域下可正确显示币种与基础配送规则。
