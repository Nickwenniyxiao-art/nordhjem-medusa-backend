# 开发者体验 (DevEx) 季度调查

> 每季度执行一次，覆盖 CI/CD 满意度、文档有用程度、部署信心、工具痛点
> 5-10 分钟完成，10 个问题以内

---

## 调查问题

### 1. CI/CD 满意度
**你对目前 CI/CD 流水线的满意度如何？**
- 1-5 分（1=非常不满意，5=非常满意）

### 2. CI 等待时间
**CI 的等待时间是否影响你的工作效率？**
- 从不 / 偶尔 / 经常 / 总是

### 3. 文档有用程度
**项目文档（RUNBOOK、架构文档、API 文档）对你的工作有多大帮助？**
- 1-5 分（1=完全没用，5=非常有用）

### 4. 部署信心
**你对部署到生产环境的信心有多大？**
- 1-5 分（1=非常没信心，5=非常有信心）

### 5. 回滚信心
**如果部署出了问题，你对回滚流程的信心有多大？**
- 1-5 分（1=完全不知道怎么回滚，5=非常有信心能快速回滚）

### 6. 最大痛点
**你在日常开发中遇到的最大痛点是什么？**（开放题）

### 7. 工具满意度
**对以下工具的满意度：**
| 工具 | 满意度 (1-5) |
|------|-------------|
| GitHub Actions CI/CD | |
| Codex (AI 编码) | |
| Playwright E2E 测试 | |
| Docker 部署 | |
| Project Board 任务管理 | |

### 8. Codex 体验
**使用 Codex 完成任务的成功率大概是多少？**
- < 30% / 30-50% / 50-70% / 70-90% / > 90%

### 9. 最想改进
**如果只能改进一件事，你最想改进什么？**（开放题）

### 10. 整体评分
**对项目整体工程体验的评分？**
- 1-10 分

---

## Google Form 创建指南

1. 打开 [Google Forms](https://forms.google.com)
2. 创建新表单，标题："NordHjem 开发者体验季度调查 — YYYY QN"
3. 按上述 10 个问题创建表单项
   - 问题 1/3/4/5/10：线性量表
   - 问题 2/8：单选
   - 问题 6/9：段落文本
   - 问题 7：网格（矩阵）
4. 开启"收集邮箱地址"（可选）
5. 发送给团队成员
