# 预算计划

## 预算目标

初期（Phase 1-2）月度运营成本控制在 **$50 以内**，优先保障交易闭环可用性与基础稳定性，避免过早投入高阶工具。

## 成本明细

| 成本项 | 月度预算 | 成本类型 | 备注 |
| --- | ---: | --- | --- |
| Railway 应用与数据库 | $12 | 固定+弹性 | 优先免费/低配，按流量上调 |
| 域名（年付摊销） | $2 | 固定 | 以 .com 或区域域名为主 |
| Stripe 手续费 | $10 | 交易相关 | 随 GMV 浮动，需纳入毛利模型 |
| AI 工具（Codex/GPT） | $20 | 固定+弹性 | 开发效率核心投入 |
| 监控与告警工具 | $3 | 可选 | 先用免费层，超限再升级 |
| 其他预留（邮件、对象存储） | $3 | 预留 | 用于突发支出 |
| **合计** | **$50** |  |  |

## 资金安排

- **Phase 1-2**：以免费/低价方案为主，投入集中在核心基础设施与支付链路。
- **Phase 3+**：根据收入与订单量调整预算，优先提升稳定性与转化相关投入（监控、CDN、营销工具）。
- **扩容原则**：只有当性能瓶颈或业务收益被量化证明后，才升级高阶资源。

## 预算控制机制

- 月度成本审查：每月固定复盘实际开销 vs 预算目标。
- 超预算告警：当月预计超过 $50 的 90% 即触发预警。
- 支出审批流程：非必要工具与升级需 Owner 审批后执行。
- 成本归因：按“基础设施/支付/工具”分类追踪，支持削减决策。
