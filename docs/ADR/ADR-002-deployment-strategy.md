# ADR-002: 部署策略

## 状态

Accepted

## 日期

2026-03

## 背景

NordHjem 需要选择部署方式。作为北欧家具电商平台，需要在成本、可控性和性能之间取得平衡。

## 决策

选择 **Docker + VPS 自托管** 方案。

### 部署架构

- 后端：Docker Compose 部署在 VPS 上
- 前端：独立部署（Next.js）
- 数据库：PostgreSQL 容器化
- CI/CD：GitHub Actions 自动构建和部署
- 分支策略：develop → staging 自动部署，main → production 需审批

### 环境配置

| 环境       | 分支    | 部署方式       | 审批       |
| ---------- | ------- | -------------- | ---------- |
| Staging    | develop | 自动           | 不需要     |
| Production | main    | 自动（审批后） | Owner 审批 |

## 替代方案

| 方案             | 优点               | 缺点                 | 结论     |
| ---------------- | ------------------ | -------------------- | -------- |
| Vercel + Railway | 零运维             | 成本高，后端限制多   | 放弃     |
| AWS ECS          | 弹性伸缩           | 复杂度高，成本不透明 | 暂不考虑 |
| VPS + Docker     | 成本可控，全栈自主 | 需要运维             | ✅ 选用  |

## 后果

- ✅ 月成本可控（固定 VPS 费用）
- ✅ 全栈可控，不受平台限制
- ⚠️ 需要自行管理备份和监控（Phase 2F 计划解决）
- ⚠️ 手动扩容（当前流量阶段不是问题）
