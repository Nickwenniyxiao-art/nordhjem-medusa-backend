# 业务规则

## 规则范围

本规则覆盖 **订单、支付、库存、定价、退换货** 五大域，适用于 NordHjem MVP 与后续迭代。

## 核心业务规则

| 规则编号 | 规则描述 | 适用场景 |
| --- | --- | --- |
| BR-001 | 订单最低支付金额为 1.00（对应币种） | 防止异常零金额订单 |
| BR-002 | 库存在支付成功后扣减，未支付订单不锁长期库存（可短时保留） | 避免支付失败导致库存错误 |
| BR-003 | 多币种价格基于主价格表+汇率快照计算，汇率每日更新一次 | EUR/USD 统一定价策略 |
| BR-004 | 退货申请期限为签收后 14 天内 | 对齐消费者保护与成本控制 |
| BR-005 | Stripe 支付成功前订单状态为 `pending`，成功后转 `paid` | 订单状态机 |
| BR-006 | 支付失败超过 3 次自动取消购物车结算会话 | 风控与资源释放 |
| BR-007 | 订单取消仅允许在未发货状态 | 售后边界 |
| BR-008 | 管理员手工改价需记录审计日志（操作者、时间、原因） | 防止价格滥用 |

## 例外处理

- Stripe 支付失败：订单保持 `pending_payment`，允许用户重试支付。
- 支付成功但回调延迟：通过 webhook 重试与后台任务补偿更新状态。
- 超卖场景：优先退款并发送解释通知，记录为库存异常事件。
- 退款失败：标记为 `refund_failed` 并进入人工处理队列。

## 规则变更流程

1. 业务规则变更由 Owner 发起或确认。
2. CTO 更新 `docs/BUSINESS-RULES.md` 并评估系统影响。
3. 同步修改代码与测试（含回归验证）。
4. 通过 PR 审核后发布，并在 CHANGELOG 记录。
